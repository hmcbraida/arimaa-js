# Arimaatic

> ⚠ Heavy work in progress.

A browser-based implementation of the [Arimaa](https://arimaa.com/arimaa/) board game with networked multiplayer, built with React, Fastify, PostgreSQL, and NATS.

## Quick start (Docker)

The fastest way to run the full stack is Docker Compose:

```bash
docker compose up --build
```

This brings up four services and runs database migrations automatically. Visit **http://localhost:8080** to play.

## Local development

### Prerequisites

- [Bun](https://bun.sh/) ≥ 1.1
- PostgreSQL 16 and NATS 2 (or use Docker for those while running the app locally)

### Install dependencies

```bash
bun install
```

### Run the SPA (offline mode only)

```bash
bun run dev
```

Starts the Vite dev server at `http://127.0.0.1:5173`. The **Offline** tab works without the API.

### Run the API server

```bash
# Set required environment variables first (see below)
bun run server
```

The server runs migrations against the configured database on startup, then listens on `PORT` (default `3001`).

### Environment variables

| Variable | Required | Default | Description |
|---|---|---|---|
| `DATABASE_URL` | Yes | -- | `postgres://user:pass@host:port/db` |
| `NATS_URL` | Yes | -- | `nats://host:4222` |
| `PORT` | No | `3001` | HTTP / WS listen port |
| `HOST` | No | `127.0.0.1` | Bind address (`0.0.0.0` in Docker) |

## Commands

| Command | Description |
|---|---|
| `bun run dev` | Start Vite dev server (SPA) |
| `bun run server` | Start Fastify API server |
| `bun run build` | Type-check + Vite production build |
| `bun run test` | Run unit / integration tests |
| `bun run test:ui` | Run Playwright end-to-end tests |
| `bun run lint` | ESLint + Biome check |
| `bun run format` | Biome auto-fix formatting |
| `bun run db:generate` | Generate a new Drizzle migration from schema changes |
| `bun run db:migrate` | Run pending migrations against `DATABASE_URL` |

## API reference

See [`docs/architecture.md`](docs/architecture.md) for the full API contract. Brief summary:

| Method | Path | Auth | Description |
|---|---|---|---|
| `POST` | `/api/sessions?side=gold\|silver` | -- | Create a session; returns `secretToken` + 8-digit `acceptToken` |
| `GET` | `/api/sessions/:id` | -- | Public snapshot of any session |
| `POST` | `/api/session-accept` | -- | Redeem an accept code; returns the joining player's `secretToken` |
| `POST` | `/api/sessions/:id/moves` | Bearer `secretToken` | Submit a move turn |
| `GET` | `/api/ws?sessionId=:id` | -- | WebSocket session event stream (upgrade) |
